# Contributing to AuRA AI

Thanks for your interest in contributing.

## Ways to Contribute

- Report bugs
- Suggest features
- Improve documentation
- Submit fixes and enhancements

## Development Setup

1. Fork the repository and clone your fork.
2. Install dependencies:
   - `npm install`
3. Start development mode:
   - `npm run dev`
4. (Optional) Build all components locally:
   - `npm run build`

Backend binaries are managed through project scripts and Make targets in the `backend` directory.

## Branching

- Create a feature branch from `main`:
  - `feat/<short-description>`
  - `fix/<short-description>`
  - `docs/<short-description>`

## Commit Guidelines

- Write clear, focused commit messages.
- Keep commits small and related to one logical change.
- Reference issues when applicable (for example: `Fixes #123`).

## Pull Request Guidelines

Before opening a pull request:

- Ensure code builds locally.
- Run linting and type checks where relevant.
- Update tests and docs when behavior changes.
- Keep PRs focused and reviewable.

When opening a pull request:

- Use the pull request template.
- Describe the problem and your solution clearly.
- Include screenshots or recordings for UI changes.
- Link related issues.

## Code Style

- Follow existing code style and architecture.
- Prefer readable and maintainable implementations.
- Avoid unrelated refactors in the same pull request.

## Reporting Bugs and Requesting Features

Please use the issue templates in this repository so maintainers can triage quickly.

## Questions

For project-related questions, open an issue or start a discussion with maintainers.

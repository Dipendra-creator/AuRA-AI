# Security Policy

## Supported Versions

Security updates are provided for the latest stable release line.

| Version | Supported |
| ------- | --------- |
| 1.0.x   | ✅ |
| < 1.0   | ❌ |

## Reporting a Vulnerability

Please do **not** report security vulnerabilities in public issues.

Instead, report privately to the maintainers via GitHub security reporting for this repository
(if enabled), or by contacting repository owners directly.

Include the following details when possible:

- A clear description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested remediation (if known)

## Response Process

Maintainers will:

1. Acknowledge receipt as quickly as possible
2. Validate and assess impact
3. Work on a fix and release plan
4. Coordinate responsible disclosure

We appreciate responsible disclosure and efforts to improve project security.
